# Эвристические методы для модифицированной задачи о назначениях

---

## Эвристические алгоритмы

### 1. Жадный алгоритм — `greedy`

На каждом шаге `j` выбирает свободную группу с максимальным значением `B[i][j]`.

**Критерий:** `argmax_{i ∉ used} B[i][j]`  
**Сложность:** O(n²)

---

### 2. χ-стратегия — `chi`

Сортирует группы по возрастанию `χᵢ` и назначает в этом порядке.  
Матрица A при принятии решений не используется.

**Критерий:** `argsort(χ)`  
**Сложность:** O(n log n)

---

### 3. Гибридная-1 — `hybrid`

Чередует два критерия:
- чётные шаги (j = 0, 2, …) → `min χᵢ` (χ-стратегия)
- нечётные шаги (j = 1, 3, …) → `argmax B[i][j]` (жадный)

**Сложность:** O(n²)

---

### 4. Гибридная-2 — `hybrid2`

Начинает с жадного шага, затем χ, и так по очереди.

- чётные шаги → `argmax B[i][j]`
- нечётные шаги → `min χᵢ`

**Сложность:** O(n²)

---

### 5. Ансамблевый жадный — `ensemble`

**Идея.** Обычный жадный: всегда выбирает лучшего из всех свободных кандидатов
и каждый раз даёт одно и то же расписание. Ансамбль решает это так же, как Random Forest
решает проблему одного дерева: запускается m независимых агентов, каждый из которых
на каждом шаге видит только случайное подмножество из `ss` свободных групп и жадно
выбирает лучшего среди них. Из-за разных случайных подмножеств агенты строят разные
расписания — результат выбирается по наибольшей итоговой прибыли `F(σ)`.

Побеждает агент с наибольшей итоговой прибылью `F(σ)`.

**Параметры:**
- `m` — количество агентов (аналог числа деревьев в Random Forest; больше → стабильнее, медленнее)
- `ss = n // 3` — размер случайной выборки кандидатов на каждом шаге

**Сложность:** O(m · n²) — формально меньше венгерского O(n³),  
но на практике работает заметно медленнее всех остальных эвристик. Причина —  
весь внутренний цикл (построение списка свободных групп, `np.random.choice`, `max` по кандидатам)  
выполняется в чистом Python и повторяется `m × n` раз.

---

### 6. Ансамблевый χ — `ensemble_chi`

Аналогично `ensemble`, но агенты выбирают `min χᵢ` среди кандидатов (без матрицы A).  
Победитель определяется по итоговой прибыли `F(σ)`.

**Параметры:** те же `m` и `ss`.  
**Сложность и замечание о скорости:** см. `ensemble` выше.

---

### 7. Двухфазная χ→жадная — `half_hybrid`

Первые `⌈n/2⌉` шагов: **χ-стратегия** — назначаем самых уязвимых групп в ранние периоды.  
Оставшиеся шаги: **онлайн-жадная** — `argmax (1−χᵢ)·A[i][j]` по текущему столбцу.

Логика: ранние периоды дают длинный хвост суммы в B — выгоднее отдать их уязвимым группам.  
Поздние периоды вносят меньший вклад, поэтому выбираем по реальному значению текущего столбца.

**Сложность:** O(n²)

---

### 8. Двухфазная жадная→χ — `half_hybrid2`

Зеркальный вариант: первые `⌈n/2⌉` шагов — онлайн-жадная, оставшиеся — χ-стратегия.

**Сложность:** O(n²)

---

## Эталон

**Венгерский алгоритм** (`hungarian`) решает задачу оптимально на полной матрице B  
с помощью `scipy.optimize.linear_sum_assignment` O(n³).  
Используется как для оценки качества эвристик.

---

## Метрики сравнения

| Метрика | Описание |
|---------|---------|
| Средняя прибыль | Среднее `F(σ)` по всем экспериментам серии |
| Относительные потери | `(F_венг − F_стратегии) / F_венг × 100%` |
| Нормализованная эффективность | Позиция стратегии в диапазоне `[худшая_в_серии, венгерский]`, % |
| Худшие потери | Максимальные потери в одном эксперименте серии |
| Время выполнения | Среднее время одного запуска, секунды |
